I have enriched all project files with detailed pedagogical comments in both English and French. The goal is to not only show what is being created, but to explain the architectural "why" behind each decision.

🎓 Educational Highlights:
Terraform Network: Detailed explanations of VPC isolation, the role of an Internet Gateway, and the strict separation between public and private subnets for security.
Terraform Compute: Explanations on how Security Groups act as virtual firewalls and how Data Sources ensure we always use the most secure, up-to-date OS images.
Terraform Database: Clarified why RDS should remain in private subnets and how Subnet Groups enable high availability across different zones.
Ansible Automation: Step-by-step breakdown of the configuration process, from package installation to service management and content deployment.
CI/CD Pipeline: Pedagogical breakdown of each stage—Linting for code quality, Checkov for security compliance, and Planning for impact analysis.
This "Starter Kit" is now a true learning resource, allowing anyone to understand modern DevSecOps patterns while they explore the code.
